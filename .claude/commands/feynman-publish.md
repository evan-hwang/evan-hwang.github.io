# 파인만 학습 블로그 발행

현재 세션의 파인만 학습 내용을 **완성도 있는 기술 블로그 글**로 변환하여 발행합니다.

## 핵심 원칙

파인만 학습 과정(Q&A, 점수, 비유 평가 등)은 **독자에게 보여주지 않는다.**
학습 과정은 입력이고, 블로그 글은 **정제된 출력**이다.

## 발행 단계

### 1. 포스트 생성

`/feynman-preview`에서 확인한 내용을 기반으로 마크다운 파일을 생성한다.

**저장 위치:**
```
docs/feynman-studies/{YYYY-MM-DD}-{topic-slug}.md
```

**파일명 규칙:**
- 파인만 학습 접두사 없이 주제 중심으로 작성
- 시리즈인 경우 끝에 번호: `{slug}-part-1.md`

**파일명 예시:**
- `2026-03-03-claude-code-skills.md`
- `2026-03-05-keda-deep-dive-part-1.md`
- `2026-03-10-keda-deep-dive-part-2.md`

### 2. 다이어그램 생성

excalidraw MCP를 사용하여 다이어그램을 생성한다.

**다이어그램 삽입 기준:**
- 아키텍처 개요: 컴포넌트 간 관계를 보여줄 때
- 데이터/제어 흐름: 요청-응답 흐름, 이벤트 파이프라인 등
- 비교 구조: Before/After, A vs B 등
- 계층 구조: 레이어, 스택 구성

**워크플로우:**
1. 미리보기에서 `[다이어그램: 설명]`으로 표시된 위치 확인
2. excalidraw MCP로 다이어그램 생성
3. 생성된 이미지를 `assets/images/feynman/`에 저장
4. 포스트에서 이미지 경로로 참조

```bash
mkdir -p assets/images/feynman
```

**이미지 파일명 규칙:**
```
assets/images/feynman/{YYYY-MM-DD}-{topic}-{n}.png
```

**포스트에서 참조:**
```markdown
![{캡션}](/assets/images/feynman/{YYYY-MM-DD}-{topic}-{n}.png)
```

### 3. 이미지 처리 (사용자 제공 이미지가 있는 경우)

사용자가 별도로 제공한 이미지가 있으면:

```bash
cp {원본 이미지 경로} assets/images/feynman/{YYYY-MM-DD}-{topic}-{n}.png
```

이미지 경로를 포스트에 반영한다.

### 4. Git 커밋 및 푸시

```bash
git add docs/feynman-studies/
git add assets/images/feynman/
git commit -m "Add post: {글 제목}

Generated with [Claude Code](https://claude.com/claude-code)

Co-Authored-By: Claude <noreply@anthropic.com>"
git push
```

{핵심 요약 2-3줄. 시리즈인 경우 다음 편 예고}

## 글쓰기 스타일 가이드

- **톤**: 전문적이지만 읽기 쉬운. 독자와 대화하듯 자연스럽게.
- **구조**: 각 섹션이 독립적으로 읽혀도 이해 가능하게.
- **근거**: 주장에는 근거를 제시. 공식 문서, 소스 코드, 벤치마크 등 인용.
- **코드**: 핵심 동작을 보여주는 최소한의 코드. 장황하지 않게.
- **비유**: 복잡한 개념은 일상적 비유로 한 번 풀어주되, 과하지 않게 1-2회만.

## 섹션 구성 유연성

위 섹션은 기본 템플릿이다. 주제에 따라 유연하게 조정한다:
- 개념 중심 주제: "배경 → 핵심 개념 → 동작 원리 → 마무리"
- 도구/프레임워크: "문제 → 아키텍처 → 컴포넌트 → 실전 → 트레이드오프"
- 비교 분석: "배경 → A vs B 구조 비교 → 성능 → 선택 기준"

## 시리즈 지원

주제가 넓어 한 편으로 다루기 어려운 경우 시리즈로 구성한다:
- 제목에 `#1`, `#2` 등을 붙인다 (예: "KEDA 딥다이브 #1: 아키텍처")
- 파일명에 `-part-1`, `-part-2`를 붙인다
- 시리즈 첫 글 도입부에 전체 로드맵을 안내한다
- 각 편 마무리에 다음 편 예고를 넣는다

## 주의사항

- 파인만 학습 과정(Q&A, 점수, 비유 평가)을 노출하지 않는다
- "나의 설명", "피드백" 같은 학습 일지 형식을 사용하지 않는다
- 학습에서 발견한 인사이트는 자연스럽게 본문에 녹인다
- 깊이가 부족한 부분은 Claude가 추가 리서치하여 보강한다
- GitHub Pages URL에 `/recoblog` prefix를 붙이지 않는다 (baseurl이 빈 문자열)
